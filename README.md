# CodePost AI 🚀🤖

An intelligent developer tool and Chrome Extension designed to automate code extraction and solution documentation for coding platforms like LeetCode. With CodePost AI, developers can instantly parse problem statements, retrieve their active solutions, and generate structured, markdown explanations, complexity analyses, and walkthroughs using advanced Large Language Models.

## Key Features

- **Automated Content Extraction**: Parses and extracts problem titles, difficulty levels, descriptions, test case examples, constraints, active programming languages, and active code directly from the web page DOM.
- **Dynamic AI Generation**: Communicates with local AI endpoints or proxy services to compose rich solution summaries, step-by-step algorithms, and time/space complexity analyses.
- **Multi-Model Support**: Configurable LLM options (such as Qwen, Kimi, GPT-OSS, and Llama models) with client-side fallback options in case of service downtime.
- **Embedded Markdown Editor**: A custom editor interface embedded directly inside the browser extension popup, allowing you to preview, refine, and format documentation before exporting or publishing.

## Architecture Overview

The project is split into two primary components:

### 1. Chrome Extension (`chrome-extension/`)
A React-based Chrome Extension built using Vite, TypeScript, TailwindCSS, and CRXJS.
- **Content Scripts**: Detects LeetCode pages, executes DOM extraction adapters, and handles script injections for sandboxed page objects (such as Monaco editor instances).
- **Background Worker**: Acts as a message broker between content scripts and the extension's popup.
- **Popup Interface**: A responsive dashboard containing markdown editors, configuration panels, and status displays.

### 2. Backend Server (`backend/`)
A Python Django service that interfaces with the Groq API to serve LLM completions.
- **REST Endpoints**: Receives parsed problem data and generates structured solution Markdown.
- **Service Layer**: Configured with robust error handling, schema validations, and configurable system instructions.

## Installation & Setup

### Prerequisites
- Node.js (version 18 or above)
- Python (version 3.12)
- Google Chrome or a Chromium-based browser

### 1. Backend Setup

1. Navigate to the backend directory:
   ```bash
   cd backend
   ```
2. Create and configure your environment file:
   - Copy `.env.example` to `.env`
   - Fill in your `GROQ_API_KEY` and other configurations:
     ```bash
     cp .env.example .env
     ```
3. Set up the virtual environment and install dependencies:
   - On Windows, run the automated script:
     ```cmd
     recreate_venv.bat
     ```
   - On macOS/Linux, run:
     ```bash
     python3 -m venv venv
     source venv/bin/activate
     pip install -r requirements.txt
     ```
4. Run database migrations:
   ```bash
   python manage.py migrate
   ```
5. Start the development server:
   ```bash
   python manage.py runserver
   ```
   The backend will be running at `http://127.0.0.1:8000/`.

### 2. Chrome Extension Setup

1. Navigate to the chrome-extension directory:
   ```bash
   cd chrome-extension
   ```
2. Create and configure your environment file:
   - Copy `.env.example` to `.env`
   - Configure the API URLs and target keys:
     ```bash
     cp .env.example .env
     ```
3. Install frontend dependencies:
   ```bash
   npm install
   ```
4. Compile and build the extension:
   - For production build:
     ```bash
     npm run build
     ```
   - For interactive development:
     ```bash
     npm run dev
     ```
5. Install the extension in Chrome:
   - Open your browser and navigate to `chrome://extensions/`
   - Enable **Developer mode** (toggle in the top-right corner)
   - Click **Load unpacked** (top-left button)
   - Select the `chrome-extension/dist` directory generated by the build process

## Configuration Variables

### Backend Configuration (`backend/.env`)
- `DJANGO_SECRET_KEY`: Private key used by Django settings for security.
- `GROQ_API_KEY`: API authentication key for Groq Cloud.
- `GROQ_MODEL`: Model ID for AI completions (defaults to `llama-3.1-8b-instant`).
- `GROQ_TEMPERATURE`: Creativity parameter (defaults to `0.2`).
- `GROQ_MAX_TOKENS`: Maximum output token limit (defaults to `4096`).

### Chrome Extension Configuration (`chrome-extension/.env`)
- `VITE_API_URL`: Backend server url (`http://localhost:8000/`).
- `VITE_APP_NAME`: Application title displayed inside the extension.
- `VITE_OPENROUTER_API_KEY`: OpenRouter API key for client-side completions.
- `VITE_OPENROUTER_BASE_URL`: Endpoint path for OpenRouter completions.
